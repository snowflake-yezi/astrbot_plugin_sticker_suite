# AstrBot 表情套件插件

群聊内学习 QQ 图片/表情包，自动标记、检索、复用，并内置 NapCat/AstrBot 表情探针。

这个新项目把原来的 `astrbot_plugin_sticker_memory` 和 `astrbot_plugin_sticker_probe` 合到了一起；旧目录暂时保留，方便回退和之后接入 git 后再逐步迁移。

## 功能

- 自动记录群聊中收到的图片/表情包组件。
- 按群隔离保存表情包数据，同时维护全局共享池用于跨群复用。
- 尽量按 `md5`、`file_id`、`file`、`path`、`url` 和本地文件内容哈希去重；同一表情只要任一强身份字段匹配已有记录，就会复用旧记录。
- 只保存带强身份字段的图片记录：`md5`、`file_id`、`file`、`path`、`url` 至少要有一个，避免把只有 `summary=表情` 的弱记录反复入库。
- 会跳过机器人自己发送的图片，避免 `表情发送` / 自动复用发出去后又被自己重新学习。
- 如果图片字段包含本地路径或可下载 URL，会缓存为本地文件用于后续发送。
- 支持查看表情列表、按编号发送、按编号打标签、删除误入库表情。
- 根据聊天文本推断心情：开心、阴阳怪气、无语、贴贴、困。
- 支持动态标签，例如：吃瓜、破防、震惊、晚安。
- 只在命中语境、冷却结束、自动复用开启时发送表情包。
- 不是每条消息随机发送，默认还有轻量概率门控，避免刷屏。

## 安装

1. 将目录放入 AstrBot 插件目录：
   ```
   AstrBot/data/plugins/astrbot_plugin_sticker_suite/
   ```
2. 重启 AstrBot。

## 指令

| 指令 | 说明 |
|------|------|
| `表情开` | 开启当前群自动复用表情包 |
| `表情关` | 关闭当前群自动复用表情包，但仍继续学习 |
| `表情跟随开` | 开启机器人回复后根据回复内容跟发表情 |
| `表情跟随关` | 关闭机器人回复跟随表情 |
| `表情跟随冷却 秒数` | 设置回复跟随发表情冷却，最低 30 秒 |
| `表情跟随测试开` | 开启 10 分钟回复跟随测试模式：跟随开启、冷却 30 秒 |
| `表情跟随测试关` | 关闭回复跟随测试模式 |
| `表情测试开` | 开启 10 分钟测试模式：自动复用开启、冷却 30 秒、触发概率 100% |
| `表情测试关` | 关闭测试模式 |
| `表情库状态` | 查看本群表情包数量、冷却和心情；跨群开启时会额外显示共享池数量和当前可用总数 |
| `表情心情` | 查看当前群机器人心情 |
| `表情冷却 秒数` | 设置当前群自动发表情冷却，最低 30 秒 |
| `表情随机` | 从当前群随机发送一张已缓存表情，用于测试发送链路 |
| `表情最近` | 发送当前群最近入库的一张已缓存表情 |
| `表情列表` | 查看当前群表情列表，按最近看到排序，每页 10 张 |
| `表情列表 页码` | 查看指定页表情列表 |
| `表情发送 编号/#ID` | 指定发送列表里的某一张表情，例如 `表情发送 1` 或 `表情发送 #A1B2C3D4` |
| `表情标记 标签` | 给最近入库的一张表情加动态标签，例如：笑死、吃瓜、破防、晚安 |
| `表情标记 编号/#ID 标签` | 给指定表情加动态标签，例如 `表情标记 2 吃瓜` |
| `表情删标 编号/#ID 标签` | 删除指定表情的某个标签，例如 `表情删标 2 吃瓜` |
| `表情清标 编号/#ID` | 清空指定表情的全部标签，例如 `表情清标 2` |
| `表情删除 编号/#ID` | 删除当前群指定表情记录，例如 `表情删除 3` |
| `表情清理重复` | 合并/移除当前群和共享池中的重复表情记录 |
| `表情自动标记开` | 开启当前群入库时自动推断标签，默认开启 |
| `表情自动标记关` | 关闭当前群自动标记，只保留手动标记 |
| `表情自动标记状态` | 查看自动标记开关、模式和推断来源 |
| `表情自动标记模式 严格/关闭` | 设置自动标记模式；严格模式会使用同条文字、同发送者上一条文字和表情元数据 |
| `表情重标记 编号/#ID` | 对已有表情重新推断标签，只追加不删除 |
| `表情重标记全部` | 对当前群全部已有表情重新推断标签 |
| `表情标签` | 查看当前群可触发标签、跨群状态和可发送缓存表情数量 |
| `表情同义词 标签 触发词` | 给标签增加动态触发词，例如 `表情同义词 生气 恼火` |
| `表情删同义词 标签 触发词` | 删除某个动态触发词，例如 `表情删同义词 生气 恼火` |
| `表情清同义词 标签` | 清空某个标签的动态触发词，例如 `表情清同义词 生气` |
| `表情同义词列表` | 查看当前群动态触发词；跨群开启时也显示共享触发词 |
| `表情跨群开` | 允许当前群使用其他群学到的表情，默认关闭；同标签优先使用本群表情 |
| `表情跨群关` | 当前群只使用本群学到的表情 |
| `表情探针状态` | 查看内置表情探针是否可用 |
| `表情探针详情` | 查看最近一次捕获到的图片/表情事件结构 |

## 工程结构

`astrbot_plugin_sticker_suite` 是新的合并项目，当前把表情记忆和表情探针放在同一个插件包里：

```text
astrbot_plugin_sticker_suite/
  main.py           # AstrBot 插件入口、命令和主流程
  constants.py      # 图片字段名、情绪词、默认冷却等共享常量
  image_extract.py  # 图片/表情识别、raw message 递归提取、source 提取
  probe.py          # 内置表情探针，输出 [sticker_probe] 诊断日志
```

旧的 `astrbot_plugin_sticker_memory` 和 `astrbot_plugin_sticker_probe` 目录暂时保留不动，方便回退。后续接入 git 后，可以继续把 `main.py` 里的标签、检索、缓存、发送等逻辑拆成独立模块。

## 表情库管理

查看表情列表：

```text
表情列表
表情列表 2
```

默认显示当前群表情；如果已经执行 `表情跨群开`，列表会显示“当前群+共享表情列表”，并把共享池里可用的表情也列出来。

列表中每一项会显示临时编号、稳定短 ID、是否可发送、身份强弱、标签、见过次数、发送次数和简短来源：

```text
1. #A1B2C3D4 [可发/强] 笑死｜见2/发1｜FC9D800F...
2. #E5F6A7B8 [未缓存/弱] 无标签｜见1/发0｜表情
```

说明：

- `编号` 会随列表排序变化，适合当场操作。
- `#短ID` 更稳定，可以用于稍后操作。
- `[强]` 表示记录里有 `md5`、`file_id`、`file`、`path`、`url` 或文件内容哈希，更容易去重。
- `[弱]` 表示只有很少字段，通常是 `summary=表情` 这类记录，更容易重复或不可发送。

按编号或短 ID 测试发送：

```text
表情发送 1
表情发送 #A1B2C3D4
```

给指定编号或短 ID 绑定标签：

```text
表情标记 1 笑死
表情标记 2 吃瓜
表情标记 #A1B2C3D4 破防
```

自动标记：

```text
表情自动标记开
表情自动标记关
表情自动标记状态
表情自动标记模式 严格
表情自动标记模式 关闭
```

自动标记开启后，表情入库时会自动追加标签。严格模式下会分析这些文本来源：

1. 同条消息文字，例如 `笑死 + 表情包`。
2. 同一发送者 60 秒内上一条文字，例如先发 `笑死我了`，再立刻补一张表情包。
3. 表情元数据文字，例如 `summary`、文件名、路径文件名中有意义的文字。
4. 直接标签、动态同义词、内置情绪词、短文本标签。

自动标记只会追加标签，不会删除手动标签。图片内容 OCR/视觉识别暂时不做，后续单独补。

对历史表情重新推断标签：

```text
表情重标记 1
表情重标记 #A1B2C3D4
表情重标记全部
```

重标记会使用表情保存过的上下文和来源摘要，只追加新标签，不会清空或覆盖已有标签。

删除某一个标签：

```text
表情删标 1 吃瓜
表情删标 #A1B2C3D4 破防
```

清空某张表情的全部标签：

```text
表情清标 1
表情清标 #A1B2C3D4
```

删除误入库的表情：

```text
表情删除 3
表情删除 #A1B2C3D4
```

清理重复记录：

```text
表情清理重复
```

`表情清理重复` 会合并当前群和共享池里的重复表情，合并依据包括强身份字段和文件内容哈希；同时会删除历史遗留的未缓存弱记录。

`表情删除 编号` 删除的是当前群的表情记录。如果这张表情没有其他群引用，也会从全局共享池中移除。

## 触发语境

内置基础语境仍然可用：

- 开心：`哈哈`、`笑死`、`绷不住`、`乐`、`草`、`好耶`
- 阴阳怪气：`急了`、`典`、`孝`、`偷笑`、`乐子`
- 无语：`无语`、`离谱`、`啊这`、`逆天`
- 贴贴：`抱抱`、`摸摸`、`可爱`、`别哭`、`贴贴`
- 困：`困`、`晚安`、`睡觉`、`累了`

也可以动态增加任意标签：

1. 先发送一张表情包。
2. 再发送 `表情标记 吃瓜`、`表情标记 破防`、`表情标记 晚安` 等，给最近入库表情打标签。
3. 如果要指定某张表情，先用 `表情列表` 看编号，再用 `表情标记 2 吃瓜`。
4. 之后聊天内容里包含这个标签文本时，就可以触发对应表情。

还可以给标签配置动态同义触发词：

```text
表情同义词 生气 恼火
表情同义词 生气 气死
表情同义词 吃瓜 细说
表情同义词列表
```

这样表情本身只需要标记 `生气`，聊天里出现 `恼火`、`气死` 也会检索到 `生气` 标签的表情。删除和清空触发词：

```text
表情删同义词 生气 恼火
表情清同义词 生气
```

自动触发现在使用本地检索评分：标签直接命中优先，其次是动态同义词、内置情绪词，再轻量参考表情曾经出现过的上下文。这个检索层也可以作为后续“机器人回复时自己带表情”的基础。

## 自动触发机制

自动触发表情需要先开启当前群自动复用：

```text
表情开
```

调试时建议使用：

```text
表情测试开
```

测试模式会持续 10 分钟，自动开启复用、把冷却设为 30 秒，并临时取消概率门控。

插件收到消息后的处理顺序：

1. 如果是机器人自己发出的消息，直接跳过，避免自己发图又被自己学习。
2. 如果消息里有可识别图片/表情包，优先学习入库，不同时触发自动发表情。
3. 如果是命令消息，例如 `/表情列表`、`表情标记`，不会触发自动发表情。
4. 普通文本消息会进入表情检索。
5. 命中候选表情后，还要经过冷却和概率门控；测试模式下概率门控临时视为通过。

检索评分优先级：

1. 直接标签命中：聊天内容里直接出现表情标签，例如 `生气`、`吃瓜`。
2. 动态同义词命中：例如配置了 `表情同义词 生气 恼火`，聊天里出现 `恼火` 会检索 `生气` 标签。
3. 内置情绪词命中：例如 `笑死`、`离谱`、`贴贴`、`晚安` 等会映射到内置标签。
4. 历史上下文轻量匹配：如果某张表情入库时保存过相似上下文，会获得较低分数的候选机会。

多个候选表情同时命中时，选择规则：

1. 分数高的优先。
2. 当前群表情优先于共享池表情。
3. 发送次数少的优先，减少重复刷同一张。
4. 最近见过的优先。
5. 仍然相同则按内部 key 稳定排序。

跨群规则：

- 默认群聊隔离，当前群不会使用共享池表情。
- 执行 `表情跨群开` 后，`表情列表`、`表情发送 编号/#ID`、`表情标记 编号/#ID 标签`、自动触发都会看到共享池。
- 如果当前群和共享池都有同名标签，优先使用当前群自己的表情。
- 当前群没有合适候选时，才使用共享池表情。

后续如果调试中改变了触发规则、选择规则、跨群规则或命令行为，需要同步更新本 README。

表情会保存到本群和全局共享池，但当前群默认不跨群取图，保持群聊表情隔离。需要跨群复用时可执行 `表情跨群开`。跨群开启后，`表情列表`、`表情发送 编号/#ID`、`表情标记 编号/#ID 标签`、自动触发都会同时看到共享池；如果 A 群和 B 群都有同名标签，会优先选择当前群自己的表情，当前群没有对应标签候选时，才使用共享池里的表情。

## 机器人回复跟随表情

普通自动触发是“用户消息命中标签/同义词后，插件主动发一张表情”。回复跟随是另一种模式：机器人先生成自己的文字回复，插件再用这段回复文本检索合适表情，并尝试把图片附加到回复结果里。

开启回复跟随：

```text
表情跟随开
```

调试时建议使用：

```text
表情跟随测试开
```

测试流程示例：

```text
表情列表
表情标记 1 阴阳怪气
表情同义词 阴阳怪气 嘲笑
表情跟随测试开
嘲笑一下我
```

预期效果是机器人先发送正常文字回复，然后回复结果里跟随一张匹配 `阴阳怪气` / `嘲笑` 语境的表情。回复跟随使用同一套检索规则：标签、动态同义词、内置情绪词和历史上下文。

可选配置：

```text
表情跟随冷却 60
表情跟随关
表情跟随测试关
```

注意：回复跟随依赖 AstrBot 运行环境提供回复结果装饰钩子。AstrBot v4.25.1 的 `result_decorate` 阶段会以 `handler(event)` 形式调用插件，因此本插件会从 event 中读取回复结果；如果日志出现 `follow skipped: no event result` 或 `follow failed: cannot append image to result`，说明当前结果对象结构还需要继续适配。

## 自动发表情测试

如果只是测试发送链路，可以用 `表情随机`、`表情最近`、`表情发送 编号`。

如果要测试“机器人在聊天里真的自动发出表情”，推荐流程：

```text
表情列表
表情标记 1 笑死
表情测试开
笑死
哈哈哈
```

测试模式持续 10 分钟，会自动开启复用、把冷却设为 30 秒，并临时取消 1/3 概率门控。测试结束后可执行：

```text
表情测试关
表情关
```

## 内置表情探针

套件已经内置原 `astrbot_plugin_sticker_probe` 的诊断能力，用来确认 NapCat/AstrBot 收到 QQ 图片、表情包时暴露的字段结构。

```text
表情探针状态
表情探针详情
```

发送图片或表情后，日志里会出现 `[sticker_probe]` 记录；`表情探针详情` 会展示最近一次捕获到的事件摘要。探针只负责诊断，不参与是否入库、是否发送、是否自动标记的业务判断。

## 说明

- 已内置表情探针，可用 `表情探针状态` / `表情探针详情` 查看 NapCat/AstrBot 暴露的图片字段。
- 如果图片字段里没有可复用 URL 或本地路径，本插件会记录元数据，但可能无法自动复用发送。
- 数据保存在插件目录下 `data/stickers.json` 和 `data/images/`。
- 当前没有额外第三方 Python 依赖，`requirements.txt` 为空是正常的。
- 重复入库会尽量合并：同一表情的 `md5`、`file_id`、`file`、`path`、`url` 或文件内容哈希任一字段匹配已有记录时，会更新旧记录而不是新增一条。
- 如果列表里仍有历史遗留的弱记录，可以执行 `表情清理重复` 删除未缓存弱记录，并合并可识别的重复记录。
- 自动发送仍受 `表情开`、冷却时间和概率门控影响；手动测试命令 `表情随机`、`表情最近`、`表情发送 编号` 不受自动复用开关影响。
- 回复跟随受 `表情跟随开` 和 `表情跟随冷却` 控制，和普通自动复用开关相互独立。
- 测试自动发送时建议使用 `表情测试开`，避免因为概率门控导致误判为没有触发。
