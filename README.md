# AstrBot 表情套件插件

群聊内学习 QQ 图片/表情包，自动标记、检索、复用，并内置 NapCat/AstrBot 表情探针。

---

## 核心理念

把"机器人发表情"从硬编码 `if 关键词` 升级成一个**可解释、可扩展的本地检索层**，让插件能：

- **学**：群里出现的图片/表情包只要带强身份字段，就自动入库。
- **记**：用文字、上下文、元数据为每张表情打标签。
- **配**：用户消息或机器人回复触发时，本地多级检索匹配最合适的一张。
- **控**：群隔离、跨群、冷却、概率门控、测试模式四个开关分别可调。

不接外部向量库、不调云端 OCR。识图使用本地 `rapidocr-onnxruntime`，默认关闭，只有执行 `表情识图开` 后才会处理无上下文表情。

---

## 功能总览

### 一、表情学习入库

- 自动识别消息里的图片/表情组件、`raw_message` 里的图片段、`picElement` 嵌套对象。
- 只保存带强身份字段的图片：`md5` / `file_id` / `file` / `path` / `url` 至少一个。`summary=表情` 这类弱记录会被忽略，避免反复入库。
- 同一张图按 `md5` / `file_id` / `file` / `path` / `url` / 本地文件 SHA-256 任一字段匹配旧记录，更新而非新增。
- 跳过机器人自己发的消息，避免自己发图又被自己学。
- 自动下载有 URL 的表情到 `data/images/`，下次可发送；下载上限 8 MB，超出跳过。
- 每张表情维护：发送/见到次数、最近见过时间、入库时间、上下文文本列表、来源摘要、强短 ID（取内容哈希前 8 位）。
- **无上下文表情的入库控制**（见下方 识图 一节）：消息没有文字、发送者 60 秒内也没有上一条文字、表情元数据也没有有意义文字——这种"纯表情"只有识图开启且冷却已过才入库；否则直接跳过。

### 二、自动打标签

入库时按"严格模式"分析三个来源给表情追加标签：

1. **同条消息文字**：图片和文字在同一条消息里发出。
2. **同发送者 60 秒内上一条文字**：典型场景"打完字立刻补一张表情"。
3. **表情元数据文字**：来源 summary、文件名、路径文件名等有意义文字（自动过滤 `[图片]` `[动画表情]` `mface` 等噪声以及一长串 md5）。

每张最多自动加 3 个标签。自动打标**只追加不删除**手动标签；手动 `表情标记` / `表情删标` / `表情清标` 永远优先。

历史表情可以批量补标：

- `表情重标记 编号/#ID`
- `表情重标记全部`

### 三、本地多级检索（核心）

用户消息或机器人回复都会过这套检索层。评分阶梯（高分优先）：

| 分数 | 命中类型 | 例子 |
|---|---|---|
| 10 | 标签字面 | 标签 `生气` ⊆ 文本 `生气了` |
| 8 | 动态同义词 | `表情同义词 生气 恼火`，文本含 `恼火` |
| 7 | 标签变体（去前/后缀） | 标签 `被欺负`，文本含 `欺负` |
| 6 | 内置语义词组 | 标签 `被欺负`，文本含 `得寸进尺` / `求饶` / `认输` |
| 5 | 内置情绪词 | 文本含 `笑死` → 标签 `笑` |
| 4 | ≥3 字标签的 2/3 字子串 | 标签 `被欺负` 的子串 `被欺`、`欺负` |
| 1–2 | 历史上下文重叠 | 当前文本和该表情入库时保存的上下文有短文本重叠 |

> 内置语义词组覆盖：被欺负 / 委屈 / 嘲笑 / 阴阳怪气 / 破防 / 无语 / 生气 / 笑 / 贴贴 / 困 / 吃瓜 / 震惊。自定义标签如果命名落在这些 key 上会自动受益。
>
> 变体 / 语义词组 / 子串只在检索时用，**不影响自动打标**。

多个候选时的选择规则：

1. 分数高优先
2. 当前群表情优先于共享池
3. `send_count` 少的优先（避免刷同一张）
4. `last_seen_at` 近的优先
5. 还相同则按内部 key 稳定排序

### 四、两种触发方式

**A. 用户消息触发（`表情开` / `表情测试开`）**

用户在群里发普通文本（不是图片、不是命令、不是机器人自己发的）→ 检索匹配 → 通过冷却和概率门控 → 自动发一张匹配表情。

**B. 机器人回复跟随（`表情跟随开` / `表情跟随测试开`）**

机器人正常生成文字回复后，挂 AstrBot `on_decorating_result` 钩子，把回复文本喂检索层 → 匹配 → 把图片**追加到回复结果链**，机器人一条消息里同时带文字 + 表情。

- 跟随会自动跳过命令触发的回复（入站消息以 `/` 或 `表情` 开头）。
  这是为了防止 `/表情标记 xxx` 的命令响应文本里包含刚被标的标签，导致跟随在命令响应上"成功"匹配并消耗冷却。
- 跟随有独立冷却 `follow_cooldown_seconds`，默认 120 秒。
- AstrBot v4.25.1 的 `result_decorate` 阶段以 `handler(event)` 形式调用，插件从 event 中读取 result。

### 五、跨群规则

- 默认群隔离：每个群只用自己学到的表情。
- 全局共享池：所有群入库时都同步进一份到 `shared.stickers`。
- `表情跨群开` 后：列表、发送、标记、自动触发都会看到共享池；同标签优先用本群表情，本群没有合适候选才用共享池。
- 同义词 `triggers` 也走类似规则：本群 + 跨群可见时才看共享池。

### 六、冷却 & 概率门控

- 用户消息触发：`cooldown_seconds`（默认 300）+ 概率门控 1/3（每分钟有效种子，避免连发）。
- 回复跟随：`follow_cooldown_seconds`（默认 120），无概率门控。
- 测试模式（10 分钟）：冷却临时改 30 秒；用户触发同时让概率门控视为通过；跟随测试模式绕过跟随冷却。

### 七、本地 OCR 识图

识图默认关闭，用于处理没有同条文字、没有同发送者上一条文字、元数据也没有有意义文本的“纯表情”。开启后，插件会用本地 `rapidocr-onnxruntime` 对已缓存图片做 OCR，识别到的文字会写入 `ocr_text`、追加到上下文，并复用现有自动打标逻辑追加标签。

- `表情识图开`：开启当前群识图。无上下文表情只有识图开启且冷却已过才会入库并排队 OCR。
- `表情识图模式 ocr/auto`：当前使用本地 OCR；`auto` 目前等价于优先 OCR。
- `表情识图模式 llm`：命令保留，但多模态 LLM 兜底暂未实现，会明确跳过。
- `表情重识图 编号/#ID`：手动对已有表情重跑 OCR，不受冷却限制。
- OCR 跑过但没识别出文字时也会推进冷却，避免无字图反复消耗资源。

### 八、内置探针

内置表情探针用于临时诊断 NapCat/AstrBot 收到 QQ 图片、表情包时暴露的字段结构。

- 探针默认关闭，避免长期记录 raw message 摘要、图片字段、URL/file/md5 等诊断信息造成日志膨胀或增加风控风险。
- 执行 `表情探针开` 后，当前群会持续开启直到手动关闭；执行 `表情探针开 10` 则只临时开启 10 分钟，过期后自动停止捕获。
- 开启期间日志前缀为 `[sticker_probe]`，会以一行 JSON 输出（含 group/sender/message_id/looks_like_image/images/event）。
- `表情探针状态`：返回开关状态、剩余时间、缓存事件数、含图事件数、最近一条 outline。
- `表情探针详情`：展示最近一次捕获到的事件结构摘要。
- 探针只负责诊断，不参与入库 / 发送 / 自动标记决策。

### 九、其它

- 数据存储：`data/stickers.json`（结构化）+ `data/images/`（本地缓存图片）。每张表情有稳定短 ID（内容 SHA-256 前 8 位），可用于 `#A1B2C3D4` 形式的命令操作。
- 重复清理：`表情清理重复` 会合并/移除当前群和共享池里 md5/file_id/url/content_hash 重叠的记录，以及无强身份且无本地缓存的弱记录。

## 安装

1. 将目录放入 AstrBot 插件目录：
   ```
   AstrBot/data/plugins/astrbot_plugin_sticker_suite/
   ```
2. 重启 AstrBot。

## 快速开始

**首次使用：**

```text
表情列表          # 看看入库情况，可能是空的
表情测试开        # 开 10 分钟测试模式，方便观察自动触发
                  # 然后让群友/自己发几张表情包
表情列表          # 应该能看到入库的表情，#短ID 列在前面
表情标记 1 笑死   # 手动标记一张做触发实验
表情同义词 笑死 哈哈  # 加同义词，让"哈哈"也能触发"笑死"
                  # 群里有人说"哈哈"或"笑死"时观察自动发图
```

**调试机器人回复跟随表情：**

```text
表情跟随测试开    # 10 分钟内冷却 30 秒、自动开启跟随
                  # 让机器人正常回复（不是命令）
                  # 看 AstrBot 日志里的 [sticker_suite] follow ... 行
表情跟随测试关
```

**自动标记示例：**

```text
表情自动标记开        # 默认就是开
                      # 群里有人发 "笑死" + 一张表情包 → 自动标 笑
                      # 群里先说 "好委屈" 60 秒内再发表情包 → 自动标 委屈
表情重标记全部        # 已有表情按现规则重新推断标签（只追加）
```

**本地 OCR 识图：**

```text
表情识图开
表情识图模式 ocr
表情重识图 1
表情列表
表情标签
```

如果图片里有文字，OCR 文本会保存到表情上下文，并尝试自动追加标签。`auto` 模式当前优先使用本地 OCR；`llm` 模式命令已保留，但多模态 LLM 兜底暂未实现。

**回退/排查：**

```text
表情探针详情      # 看 raw_message 里的图片字段长什么样
表情库状态        # 查冷却、跟随、自动标记、跨群开关、心情
表情清理重复      # 合并去重
表情删除 #A1B2C3D4   # 误入库可按短 ID 删
```

## 命令速查

下面所有 `表情...` 指令都支持直接发送，也兼容 AstrBot 的 `/` 前缀。例如 `表情库状态` 和 `/表情库状态` 都可以触发；探针短命令同理支持 `探针开`、`/探针开`。

| 指令 | 说明 |
|------|------|
| `表情开` | 开启当前群自动复用表情包 |
| `表情关` | 关闭当前群自动复用表情包，但仍继续学习 |
| `表情跟随开` | 开启机器人回复后根据回复内容跟发表情 |
| `表情跟随关` | 关闭机器人回复跟随表情 |
| `表情跟随冷却 秒数` | 设置回复跟随发表情冷却，最低 30 秒 |
| `表情跟随测试开` | 开启 10 分钟回复跟随测试模式：跟随开启、冷却 30 秒 |
| `表情跟随测试关` | 关闭回复跟随测试模式 |
| `表情测试开` | 开启 10 分钟测试模式：自动复用开启、冷却 30 秒、触发概率 100% |
| `表情测试关` | 关闭测试模式 |
| `表情库状态` | 查看本群表情包数量、冷却和心情；跨群开启时会额外显示共享池数量和当前可用总数 |
| `表情心情` | 查看当前群机器人心情 |
| `表情冷却 秒数` | 设置当前群自动发表情冷却，最低 30 秒 |
| `表情随机` | 从当前群随机发送一张已缓存表情，用于测试发送链路 |
| `表情最近` | 发送当前群最近入库的一张已缓存表情 |
| `表情列表` | 查看当前群表情列表，按最近看到排序，每页 10 张 |
| `表情列表 页码` | 查看指定页表情列表 |
| `表情详情 编号/#ID` | 查看指定表情的详细信息，包括标签、上下文、OCR、来源摘要和统计信息 |
| `表情发送 编号/#ID` | 指定发送列表里的某一张表情，例如 `表情发送 1` 或 `表情发送 #A1B2C3D4` |
| `表情标记 标签` | 给最近入库的一张表情加动态标签，例如：笑死、吃瓜、破防、晚安 |
| `表情标记 编号/#ID 标签` | 给指定表情加动态标签，例如 `表情标记 2 吃瓜` |
| `表情删标 编号/#ID 标签` | 删除指定表情的某个标签，例如 `表情删标 2 吃瓜` |
| `表情清标 编号/#ID` | 清空指定表情的全部标签，例如 `表情清标 2` |
| `表情删除 编号/#ID` | 删除当前群指定表情记录，例如 `表情删除 3` |
| `表情清理重复` | 合并/移除当前群和共享池中的重复表情记录 |
| `表情自动标记开` | 开启当前群入库时自动推断标签，默认开启 |
| `表情自动标记关` | 关闭当前群自动标记，只保留手动标记 |
| `表情自动标记状态` | 查看自动标记开关、模式和推断来源 |
| `表情自动标记模式 严格/关闭` | 设置自动标记模式；严格模式会使用同条文字、同发送者上一条文字和表情元数据 |
| `表情重标记 编号/#ID` | 对已有表情重新推断标签，只追加不删除 |
| `表情重标记全部` | 对当前群全部已有表情重新推断标签 |
| `表情识图开` | 开启识图，无上下文表情才会入库（受冷却约束） |
| `表情识图关` | 关闭识图，无上下文表情直接跳过入库（默认） |
| `表情识图模式 ocr/llm/auto` | 选择识图模式；`ocr`/`auto` 使用本地 OCR，`llm` 暂未实现 |
| `表情识图冷却 分钟数` | 设置识图冷却（无上下文表情触发后 N 分钟内不再识图），范围 1–1440，默认 10 |
| `表情识图状态` | 查看识图开关、模式、冷却剩余时间、已识图表情数 |
| `表情重识图 编号/#ID` | 手动对指定表情重跑识图，不受冷却约束 |
| `表情重识图全部` | 手动对当前群全部表情重跑识图 |
| `表情标签` | 查看当前群可触发标签、跨群状态和可发送缓存表情数量 |

| `表情同义词列表` | 查看当前群动态触发词；跨群开启时也显示共享触发词 |

| `表情跨群关` | 当前群只使用本群学到的表情 |
| `表情探针开` / `探针开` | 开启当前群表情探针，不自动关闭，期间输出 `[sticker_probe]` 诊断日志 |
| `表情探针开 分钟数` / `探针开 分钟数` | 临时开启当前群表情探针 N 分钟，范围 1–1440 |
| `表情探针关` | 关闭当前群表情探针 |
| `表情探针状态` | 查看内置表情探针是否开启、剩余时间和缓存事件数 |
| `表情探针详情` | 查看最近一次捕获到的图片/表情事件结构 |

## 表情管理与详情

查看表情列表：

```text
表情列表
表情列表 2
```

默认显示当前群表情；如果已经执行 `表情跨群开`，列表会显示“当前群+共享表情列表”，并把共享池里可用的表情也列出来。

列表中每一项会显示临时编号、稳定短 ID、是否可发送、身份强弱、标签、见过次数、发送次数和简短来源：

```text
1. #A1B2C3D4 [可发/强] 笑死｜见2/发1｜FC9D800F...
2. #E5F6A7B8 [未缓存/弱] 无标签｜见1/发0｜表情
```

说明：

- `编号` 会随列表排序变化，适合当场操作。
- `#短ID` 更稳定，可以用于稍后操作。
- `[强]` 表示记录里有 `md5`、`file_id`、`file`、`path`、`url` 或文件内容哈希，更容易去重。
- `[弱]` 表示只有很少字段，通常是 `summary=表情` 这类记录，更容易重复或不可发送。

按编号或短 ID 查看单张表情详情：

```text
表情详情 1
表情详情 #A1B2C3D4
```

详情会尽量展示这张表情当前已有的标签、上下文、OCR 文本、来源摘要、发送/见到次数、时间等信息；具体字段取决于这条记录本身是否已有对应元数据。

按编号或短 ID 测试发送：

```text
表情发送 1
表情发送 #A1B2C3D4
```

给指定编号或短 ID 绑定标签：

```text
表情标记 1 笑死
表情标记 2 吃瓜
表情标记 #A1B2C3D4 破防
```

自动标记：

```text
表情自动标记开
表情自动标记关
表情自动标记状态
表情自动标记模式 严格
表情自动标记模式 关闭
```

自动标记开启后，表情入库时会自动追加标签。严格模式下会分析这些文本来源：

1. 同条消息文字，例如 `笑死 + 表情包`。
2. 同一发送者 60 秒内上一条文字，例如先发 `笑死我了`，再立刻补一张表情包。
3. 表情元数据文字，例如 `summary`、文件名、路径文件名中有意义的文字。
4. 直接标签、动态同义词、内置情绪词、短文本标签。

自动标记只会追加标签，不会删除手动标签。图片内容 OCR 需要开启 `表情识图开`，识别到的文字会作为额外上下文参与自动打标。

对历史表情重新推断标签：

```text
表情重标记 1
表情重标记 #A1B2C3D4
表情重标记全部
```

重标记会使用表情保存过的上下文和来源摘要，只追加新标签，不会清空或覆盖已有标签。

删除某一个标签：

```text
表情删标 1 吃瓜
表情删标 #A1B2C3D4 破防
```

清空某张表情的全部标签：

```text
表情清标 1
表情清标 #A1B2C3D4
```

删除误入库的表情：

```text
表情删除 3
表情删除 #A1B2C3D4
```

清理重复记录：

```text
表情清理重复
```

`表情清理重复` 会合并当前群和共享池里的重复表情，合并依据包括强身份字段和文件内容哈希；同时会删除历史遗留的未缓存弱记录。

`表情删除 编号` 删除的是当前群的表情记录。如果这张表情没有其他群引用，也会从全局共享池中移除。

## 标签与触发词

内置基础语境仍然可用：

- 开心：`哈哈`、`笑死`、`绷不住`、`乐`、`草`、`好耶`
- 阴阳怪气：`急了`、`典`、`孝`、`偷笑`、`乐子`
- 无语：`无语`、`离谱`、`啊这`、`逆天`
- 贴贴：`抱抱`、`摸摸`、`可爱`、`别哭`、`贴贴`
- 困：`困`、`晚安`、`睡觉`、`累了`

也可以动态增加任意标签：

1. 先发送一张表情包。
2. 再发送 `表情标记 吃瓜`、`表情标记 破防`、`表情标记 晚安` 等，给最近入库表情打标签。
3. 如果要指定某张表情，先用 `表情列表` 看编号，再用 `表情标记 2 吃瓜`。
4. 之后聊天内容里包含这个标签文本时，就可以触发对应表情。

还可以给标签配置动态同义触发词：

```text
表情同义词 生气 恼火
表情同义词 生气 气死
表情同义词 吃瓜 细说
表情同义词列表
```

这样表情本身只需要标记 `生气`，聊天里出现 `恼火`、`气死` 也会检索到 `生气` 标签的表情。删除和清空触发词：

```text
表情删同义词 生气 恼火
表情清同义词 生气
```

自动触发现在使用本地检索评分：标签直接命中优先，其次是动态同义词、内置情绪词，再轻量参考表情曾经出现过的上下文。这个检索层也可以作为后续“机器人回复时自己带表情”的基础。

## 自动触发与检索

自动触发表情需要先开启当前群自动复用：

```text
表情开
```

调试时建议使用：

```text
表情测试开
```

测试模式会持续 10 分钟，自动开启复用、把冷却设为 30 秒，并临时取消概率门控。

插件收到消息后的处理顺序：

1. 如果是机器人自己发出的消息，直接跳过，避免自己发图又被自己学习。
2. 如果消息里有可识别图片/表情包，优先学习入库，不同时触发自动发表情。
3. 如果是命令消息，例如 `/表情列表`、`表情标记`，不会触发自动发表情。
4. 普通文本消息会进入表情检索。
5. 命中候选表情后，还要经过冷却和概率门控；测试模式下概率门控临时视为通过。

检索评分优先级（分数高的优先）：

1. **标签字面命中（10 分）**：聊天/回复里直接出现表情标签，例如标签 `生气` 命中文本里的 "生气"。
2. **动态同义词命中（8 分）**：例如配置了 `表情同义词 生气 恼火`，文本里出现 `恼火` 会算 `生气` 标签命中。
3. **标签变体命中（7 分）**：把常见前缀（被/不/没/小/老）或后缀（吧/啊/啦/呢/了/的）剥掉后再字面匹配。例如标签 `被欺负` 也能命中文本里的 `欺负`、标签 `生气` 也能命中 `生气了`。
4. **内置语义词组命中（6 分）**：内置常见标签家族（被欺负 / 委屈 / 嘲笑 / 阴阳怪气 / 破防 / 无语 / 生气 / 笑 / 贴贴 / 困 / 吃瓜 / 震惊）的语义近义词。例如标签 `被欺负` 会被文本里 `得寸进尺`、`认输`、`投降`、`求饶` 等语义词触发。自定义标签如果名字落在家族 key 上会自动受益。
5. **内置情绪词命中（5 分）**：例如 `笑死`、`离谱`、`贴贴`、`晚安` 映射到内置情绪标签 `笑`、`无语`、`贴贴`、`困`。
6. **标签连续 2/3 字子串命中（4 分）**：仅当标签 ≥ 3 字时启用，例如标签 `被欺负` 的子串 `被欺`、`欺负`。2 字标签不做子串拆分，避免误命中。
7. **历史上下文轻量匹配**：如果某张表情入库时保存过相似上下文，会获得较低分数的候选机会。

> 检索侧的"变体 / 语义词组 / 子串"只用于匹配，不会影响自动打标——自动打标仍然只用字面、同义词、内置情绪词、短文本。

多个候选表情同时命中时，选择规则：

1. 分数高的优先。
2. 当前群表情优先于共享池表情。
3. 发送次数少的优先，减少重复刷同一张。
4. 最近见过的优先。
5. 仍然相同则按内部 key 稳定排序。

跨群规则：

- 默认群聊隔离，当前群不会使用共享池表情。
- 执行 `表情跨群开` 后，`表情列表`、`表情发送 编号/#ID`、`表情标记 编号/#ID 标签`、自动触发都会看到共享池。
- 如果当前群和共享池都有同名标签，优先使用当前群自己的表情。
- 当前群没有合适候选时，才使用共享池表情。

后续如果调试中改变了触发规则、选择规则、跨群规则或命令行为，需要同步更新本 README。

## 回复跟随

普通自动触发是“用户消息命中标签/同义词后，插件主动发一张表情”。回复跟随是另一种模式：机器人先生成自己的文字回复，插件再用这段回复文本检索合适表情，并尝试把图片附加到回复结果里。

开启回复跟随：

```text
表情跟随开
```

调试时建议使用：

```text
表情跟随测试开
```

测试流程示例：

```text
表情列表
表情标记 1 阴阳怪气
表情同义词 阴阳怪气 嘲笑
表情跟随测试开
嘲笑一下我
```

预期效果是机器人先发送正常文字回复，然后回复结果里跟随一张匹配 `阴阳怪气` / `嘲笑` 语境的表情。回复跟随使用同一套检索规则：标签、动态同义词、内置情绪词和历史上下文。

可选配置：

```text
表情跟随冷却 60
表情跟随关
表情跟随测试关
```

注意：回复跟随依赖 AstrBot 运行环境提供回复结果装饰钩子。AstrBot v4.25.1 的 `result_decorate` 阶段会以 `handler(event)` 形式调用插件，因此本插件会从 event 中读取回复结果；如果日志出现 `follow skipped: no event result` 或 `follow failed: cannot append image to result`，说明当前结果对象结构还需要继续适配。

## 测试与排查

如果只是测试发送链路，可以用 `表情随机`、`表情最近`、`表情发送 编号`。

如果要测试“机器人在聊天里真的自动发出表情”，推荐流程：

```text
表情列表
表情标记 1 笑死
表情测试开
笑死
哈哈哈
```

测试模式持续 10 分钟，会自动开启复用、把冷却设为 30 秒，并临时取消 1/3 概率门控。测试结束后可执行：

```text
表情测试关
表情关
```

## 内置表情探针

套件内置表情探针诊断能力，用来确认 NapCat/AstrBot 收到 QQ 图片、表情包时暴露的字段结构。

```text
表情探针开        # 一直开启，直到手动关闭
探针开            # 同上，短命令
表情探针开 10     # 只开启 10 分钟
探针开 10         # 同上，短命令
表情探针状态
表情探针详情
表情探针关
```

探针默认关闭。执行 `表情探针开` / `探针开` 后，当前群会一直捕获事件结构直到手动关闭；执行 `表情探针开 10` / `探针开 10` 则只临时开启 10 分钟。开启后日志里才会出现 `[sticker_probe]` 记录；`表情探针详情` 会展示最近一次捕获到的事件摘要。探针只负责诊断，不参与是否入库、是否发送、是否自动标记的业务判断，不建议长期开启。

## 限制与说明

- 已内置表情探针，但默认关闭；用 `表情探针开` / `探针开` 持续开启，或用 `表情探针开 10` / `探针开 10` 临时开启后，才会输出 `[sticker_probe]` 日志并缓存事件结构。
- 如果图片字段里没有可复用 URL 或本地路径，本插件会记录元数据，但可能无法自动复用发送。
- 数据保存在插件目录下 `data/stickers.json` 和 `data/images/`。
- 识图依赖 `rapidocr-onnxruntime`，已写入 `requirements.txt`；如果依赖未安装，插件仍能加载，但执行识图时会提示依赖缺失。
- 重复入库会尽量合并：同一表情的 `md5`、`file_id`、`file`、`path`、`url` 或文件内容哈希任一字段匹配已有记录时，会更新旧记录而不是新增一条。
- 如果列表里仍有历史遗留的弱记录，可以执行 `表情清理重复` 删除未缓存弱记录，并合并可识别的重复记录。
- 自动发送仍受 `表情开`、冷却时间和概率门控影响；手动测试命令 `表情随机`、`表情最近`、`表情发送 编号` 不受自动复用开关影响。
- 回复跟随受 `表情跟随开` 和 `表情跟随冷却` 控制，和普通自动复用开关相互独立。
- 测试自动发送时建议使用 `表情测试开`，避免因为概率门控导致误判为没有触发。

## 工程结构

当前文件分工如下：

```text
astrbot_plugin_sticker_suite/
  __init__.py        # 暴露 StickerSuitePlugin
  main.py            # AstrBot 装饰器注册、命令编排、数据读写
  constants.py       # 共享常量：图片字段名、强身份字段、情绪词、默认冷却等
  image_extract.py   # 消息组件/raw message 递归识别为可入库 image 记录
  probe.py           # 内置 [sticker_probe] 诊断
  vision.py          # 本地 OCR / 识图入口
  requirements.txt   # 可选识图依赖
  metadata.yaml
  README.md
  CLAUDE.md
```

- 类名是 `StickerSuitePlugin`，日志前缀是 `[sticker_suite]`（探针日志仍是 `[sticker_probe]`）。
- 回复跟随依赖 AstrBot `filter.on_decorating_result`；若运行环境没有该钩子，启动日志会打印一条 `filter.on_decorating_result not available` 警告。
